# タバネル Codex指示書

このリポジトリで作業またはレビューを行う前に、必ず以下を読むこと。

1. `AI_COLLABORATION_GUIDE.md`
2. `CLAUDE.md`
3. 対象Pull Requestの目的、変更範囲、完了条件

## Codexの標準役割

- アプリ本体の変更では、Claude Codeが実装者、Codexが独立レビュー担当である。
- LP、調査、販促、KPI設計では、オーナーの指定によりCodexが実装者になる場合がある。
- レビュー依頼ではコードを変更しない。オーナーから修正を明示された場合だけ変更する。
- 実装者の説明を追認せず、実際の差分と関連コードから判断する。
- オーナーの明示的な「公開OK」なしに、`main`へのマージ、本番公開、外部送信を行わない。

## Review guidelines

Pull Requestレビューでは、重要度の高い具体的な問題だけを報告する。根拠のない可能性や好みの違いは指摘しない。

特に以下を確認すること。

- `localStorage`キー`genba_note_v1`と既存データの互換性
- JSONバックアップの書き出し・読み込み互換性
- 顧客、案件、メニュー、事業者情報の消失・破損リスク
- 入力値のエスケープ漏れ、DOM XSS、危険なHTML挿入
- 顧客名、電話番号、住所、案件メモの意図しない外部送信
- 匿名統計が明示同意時だけ送信され、画面説明と送信内容が一致していること
- 見積書、請求書、領収書の小計、税額、合計、状態変更の正確性
- スマートフォンで主要操作が完結し、行き止まりや誤操作を生まないこと
- LPや画面の訴求と実際の機能・データ取扱いが一致していること
- APIキー、トークン、ダッシュボードの鍵などの秘密情報が含まれていないこと
- 変更目的に対して差分が過大でなく、Phase 0の利用者検証を妨げる機能過多でないこと
- オーナー承認が必要な操作を自動化・実行していないこと

以下はP1相当として扱う。

- 既存利用者のデータ消失または読み込み不能
- 顧客情報や秘密情報の漏えい
- 金額、税額、請求状態の重大な誤り
- アプリが起動しない、保存できない、主要操作が完了できない回帰
- オーナー承認を経ないマージ、公開、外部送信、破壊的変更

レビューコメントには、問題が起きる条件、影響、該当箇所、最小限の修正方針を含めること。問題がなければ、確認した範囲と未確認事項を簡潔に記載すること。

## 最小検証

変更に応じて、少なくとも以下を確認すること。

- インラインJavaScriptの構文確認
- 主要な保存・再読み込み動作
- バックアップの書き出し・読み込み
- 見積書、請求書、領収書の計算と印刷表示
- スマートフォン幅での主要導線
- 変更した外部URLの応答

実施していない検証を「確認済み」と報告しないこと。
