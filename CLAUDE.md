# タバネル(TABANERU)プロジェクト引き継ぎ書

出張・訪問サービスの個人事業主向け「ひとり業務管理」無料Webアプリ。
オーナー(ユーザー)は非エンジニア。営業・マーケ・接客が得意。説明は専門用語を避け平易に。

## AI共同開発の必須ルール

- 作業前に`AI_COLLABORATION_GUIDE.md`を必ず読む。**実装は常にClaude、独立レビューは常にCodex（レビュー専任）**とする。
- タバネルの相談窓口は常にClaude。オーナーと構想を練った同じClaudeセッションで、仕様確定・実装・PR作成・Codexレビュー対応・公開報告まで維持する。
- PR作成後も元のセッションを終了せず、Codexのレビューを定期確認する。
- GitHub上の`@claude`やGitHubイベント起点のRoutineで、別のClaudeを新規起動しない。
- Codexから指摘があれば同じセッションで修正し、再レビューを待つ。自動修正は最大2回とする。
- 2回で承認に至らない場合は、論点と選択肢を整理してオーナーへ判断を求める。
- Codexが承認しても自動マージしない。オーナーの「公開OK」後にマージ・公開する。
- 元のセッションを閉じた場合は、新規チャットではなくContinue／Resumeで再開する。

## 役割プロトコル
- **フェイブル**(claude-fable-5): 戦略立案・意思決定の壁打ち
- **オーパス**(claude-opus-4-8): 機能開発・複雑な実装
- **サン**(claude-sonnet-5): 文言・投稿文・軽作業
- ユーザーが名前で呼んだらその役として振る舞う。返答は簡潔に。
- セッション終了前に本ファイルの「現在地」を更新すること。

## 戦略の全体像
**POD(差別化)**: 「予定・顧客・請求の一体化(束ねる)」。書類作成専業アプリ(ジムー等)とは棚が違う。
競合レビュー調査で判明した不満(広告・透かし・登録地獄・データ消失・法人向けは高額)の裏返しがブランドの約束。

| フェーズ | 目標 | 収益 |
|---|---|---|
| **Phase 0(今)** | 週1回使う人を10人 | ¥0。無料で信頼を配る |
| Phase 1 | 利用者の声で改良+ウェイティングリスト蓄積 | 買い切りPro(書類カスタム等)を検討 |
| Phase 2 | クラウド同期(月480円想定)開始 | ホスティングをCloudflare等へ移転(GitHub Pagesは商用SaaS不可)。独自ドメインを移転前に取得 |

**KPIと判断基準**
- 最重要: 週次アクティブ(GoatCounterの app-open-return / 統計ダッシュボードの継続週数)
- 有料GO判定: アプリ利用者のうちウェイティングリスト登録が**10%以上**ならクラウド同期の開発着手。1%未満なら訴求・価格を再考
- フォームの「月額いくらまで」回答が価格設定の根拠

**集客方針**
- 主軸: X(@tabaneru_jp)での発信+オーナーの人脈への直接共有(LINE)
- 投稿3本柱: ①個人事業主あるある(共感) ②実務お役立ち(信頼) ③機能紹介(製品)
- 運用: サンが投稿文を量産→オーナーがXの公式予約投稿にセット。**自動DM・自動一括操作は絶対禁止**
- SEOは長期戦のため当面期待しない。URL直接配布が基本

- `app/index.html` — アプリ本体(単一HTML・vanilla JS・バックエンドなし)

## 開発フロー
1. オーナーと同じClaudeセッションで構想・仕様・完了条件を確定する
2. `AI_COLLABORATION_GUIDE.md`と作業中PRを確認する
3. `git fetch origin main && git checkout -B claude/<task-name> origin/main`
4. 合意した仕様をPR本文用に整理してから実装する
5. 実装 → Playwrightで検証する
6. コミット→プッシュ→PR作成 → 同じセッションでCodexレビューを確認する
7. 指摘があれば最大2回まで修正・再レビューする
8. Codex承認後、オーナーへ変更点・検証結果・残存リスクを報告する
9. **オーナーの「公開OK」を待ってから**マージ・本番確認する

Playwrightは利用可能な実行環境とブラウザパスを開始時に確認する。GitHub Pagesのデプロイが一時失敗した場合も、原因を確認せず空コミットで再実行しない。

## 絶対に守ること(ブランドの約束)
- 基本機能はずっと無料 / 広告なし / 登録不要 / 書類に透かしなし / 圏外でも動く
- 利用者の顧客情報・売上を運営側に送らない(計測は匿名の訪問数と同意制の統計のみ)

- 破壊的操作・外部公開・課金関連は必ずユーザーの承認を得る
- 本リポジトリは公開。秘密情報(APIキー・ダッシュボードの合鍵等)を絶対にコミットしない
