# タバネル(TABANERU)プロジェクト引き継ぎ書

請求書を発行する"ひとり事業者"向け「ひとり業務管理」無料Webアプリ。
(2026-07-08にターゲットを「出張・訪問サービス限定」から「請求書を出す個人事業主全般」に拡大。出張・訪問系は業種特化版=Phase 3の本命として残す)
オーナー(ユーザー)は非エンジニア。営業・マーケ・接客が得意。**LINE人脈・増幅役・広告予算はない**(2026-07-08確認)。説明は専門用語を避け平易に。

## 役割プロトコル
- **フェイブル**(claude-fable-5): 戦略立案・意思決定の壁打ち
- **オーパス**(claude-opus-4-8): 機能開発・複雑な実装
- **サン**(claude-sonnet-5): 文言・投稿文・軽作業
- ユーザーが名前で呼んだらその役として振る舞う。返答は簡潔に。
- セッション終了前に本ファイルの「現在地」を更新すること。

## 戦略の全体像
**POD(差別化)**: 「予定・顧客・請求の一体化(束ねる)」。書類作成専業アプリ(ジムー等)とは棚が違う。
競合レビュー調査で判明した不満(広告・透かし・登録地獄・データ消失・法人向けは高額)の裏返しがブランドの約束。

| フェーズ | 目標 | 収益 |
|---|---|---|
| **Phase 0(今)** | 週1回使う人を10人 | ¥0。無料で信頼を配る |
| Phase 1 | 利用者の声で改良+ウェイティングリスト蓄積 | 買い切りPro(書類カスタム等)を検討 |
| Phase 2 | クラウド同期(月480円想定)開始 | ホスティングをCloudflare等へ移転(GitHub Pagesは商用SaaS不可)。独自ドメインを移転前に取得。**有料の目玉候補: カレンダー連携(Googleカレンダー等)** — オーナー発案。要OAuth+サーバー=Phase 2の土台が前提。ウェイティングリストでニーズ検証してから作る |
| Phase 3 | 業種特化版・導入サポート | 単価アップ |

**KPIと判断基準**
- 最重要: 週次アクティブ(GoatCounterの app-open-return / 統計ダッシュボードの継続週数)
- 有料GO判定: アプリ利用者のうちウェイティングリスト登録が**10%以上**ならクラウド同期の開発着手。1%未満なら訴求・価格を再考
- フォームの「月額いくらまで」回答が価格設定の根拠

**集客方針(2026-07-08改訂・事例調査2本で裏取り済み)**
- **SNSはX一点集中**。根拠: 初期成長はほぼ1〜2チャネル(Lenny Rachitsky 100社調査)/国内実例MENTA・ZennもX一点/ひとり運営の複数SNS並行の成功反証なし。20〜40代のX利用率48〜78%(総務省R6調査)
- 投稿3本柱: ①個人事業主あるある(共感) ②実務お役立ち(信頼) ③機能紹介(製品)。**毎回"一人の具体的ペルソナ"に向けて書く**(今週は撮影フリーランス、来週は整体…)。狙いは広く、言葉は狭く
- 運用: サンが投稿文を量産→オーナーがSocialDog(無料枠: 同時予約10件・分析2週間)にセット。**リプ・会話は必ず人が返す**(バズより定着。バズは定着に直結しないという個人開発者の証言あり)。**自動DM・自動一括操作は絶対禁止**
- **人脈はコミュニティでゼロから作る**: フリーランス協会(会員13.3万人・無料会員あり)、FreelanceNow(FBグループ・無償)に、ひとり事業者として参加。売り込まず悩みを共有、プロフィールにタバネルを置く。仕事獲得経路1位は人脈72.8%(フリーランス白書2025)
- **Instagramは第2章**: 美容・整体・撮影系の生息地はInstagram(ネイルは来店の7割超がSNS経由)。Xで刺さる型が確立してから開設。今は作らない
- **SEO・広告はPhase 2まで封印**: 検索面はfreee/弥生/マネフォがシェア93.7%で寡占。広告はLTV=0の今は経済合理性なし(Misocaも広告は"件数取れず除外")。売る物ができたらfreee型で投入
- 4週ごとにGoatCounter・週次アクティブで振り返り。Xが空振りならInstagramへ軸足を移す

8. (保留) tabaneru.appドメイン取得(集客本格化の直前でよい)/ 統計ダッシュボードの旧名表記修正(次回Cloudflareトークン発行時)

## プロダクト構成
- `app/index.html` — アプリ本体(単一HTML・vanilla JS・バックエンドなし)
  - データは利用者端末のlocalStorage(キー `genba_note_v1` — **互換のため変更禁止**)
  - 機能: 予定/顧客/見積・請求・領収書/まとめ請求/LINEメッセージ生成/売上チャート/検索/バックアップ
- `lp/index.html` — 紹介LP(+ `lp/ogp.png` OGP画像)
- `telemetry/` — Cloudflare Worker(同意制の匿名利用統計。D1保存)
- `brand/` — ブランド素材(Xアイコン・ヘッダーPNGと、その生成元HTML。色や文言を変えて再生成可能)
- ルートの `index.html` はProgate練習ページ(触らない)

## 開発フロー
1. `git fetch origin main && git checkout -B claude/revenue-examples-k3to5g origin/main`(マージ済みなら作り直し)
2. 実装 → Playwright(グローバル導入済み・executablePath: /opt/pw-browsers/chromium-1194/chrome-linux/chrome)で検証
3. コミット→プッシュ→PR作成 → **ユーザーの「公開OK」を待ってから**マージ
4. GitHub Pagesのデプロイは稀に一時失敗する(mainへの空コミットで再実行)

## 絶対に守ること(ブランドの約束)
- 基本機能はずっと無料 / 広告なし / 登録不要 / 書類に透かしなし / 圏外でも動く
